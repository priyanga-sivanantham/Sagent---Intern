## Authentication Details
The backend uses JWT.

Register:
- `POST /users/register`

Login:
- `POST /users/login`

Login response is:
```json
{
  "token": "JWT_TOKEN",
  "tokenType": "Bearer",
  "expiresInSeconds": 3600,
  "expiresAt": "2026-03-23T17:25:05Z",
  "user": {
    "userId": 1,
    "name": "Organizer",
    "email": "organizer@example.com",
    "phone": "9999999999",
    "role": "ORGANIZER"
  }
}
```

Also available:
- `GET /users/me`
- `POST /users/refresh`

Requirements:
- Store JWT token in localStorage
- Store logged-in user in auth context
- On app load, restore token from localStorage
- If token exists, call `/users/me`
- If session is invalid, clear auth and redirect to `/login`
- Attach `Authorization: Bearer <token>` on protected API calls
- Handle 401 errors globally

## Important Notes About API Responses

```json
  "totalItems": 1,
  "totalPages": 1

```
So build frontend components flexible enough to handle both plain arrays and paginated responses where needed.
